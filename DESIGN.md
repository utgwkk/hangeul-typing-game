# ハングルタイピングゲーム デザインドキュメント

## Context（なぜ作るか）

ハングル（韓国語）キーボード入力を練習するためのタイピングゲームが欲しい。
Web ブラウザ上で **サーバーなし（HTML + JavaScript + CSS のみの静的ファイル）** で完結させたい。
韓国語キーボード（2벌식 / 두벌식, Dubeolsik）の指の位置を、OS の韓国語 IME を別途設定しなくても学べるようにすることが狙い。

### 確定した要件
| 項目 | 決定 |
|------|------|
| 入力方式 | **JS 内で字母（자모）を自前合成**。OS の IME を使わず、物理キー(QWERTY 位置) → 2벌식の字母 → 音節へ JS で組み立てる |
| UI 言語 | **日本語 / 韓国語の切替**。i18n ライブラリを使用 |
| 練習レベル | **音節(글자) / 単語(단어) / 文章(문장)** の 3 モード（字母単独モードは v1 では持たない） |
| 補助機能 | **画面上キーボード表示 / 統計(WPM・正確度) / スコア・タイマー** |
| 非対象(v1) | 結果の永続化(localStorage 保存・履歴・ベストスコア) |

---

## スコープ

### v1 でやること
- 2벌식レイアウトの自前入力オートマトン（IME 相当の合成処理）
- 3 つの練習モード（音節 / 単語 / 文章）
- 画面上の仮想キーボード（次に押すキーのハイライト、押下フィードバック）
- リアルタイム統計（速度・正確度）とスコア / タイマー
- 日本語 ⇄ 韓国語の UI 切替（i18n）

### v1 でやらないこと（将来拡張）
- ベストスコア / 履歴の localStorage 保存
- 字母単独の基礎ドリル、運指レッスン（段階カリキュラム）
- 3벌식など他レイアウト対応
- 効果音 / BGM、ランキング・共有

## 技術選定

- **Vite + TypeScript + React**。`vite build` で生成される成果物は **静的ファイル（HTML + JS + CSS）のみ** なので、「サーバーなしで動作」という要件を満たす（実行時バックエンド不要、静的ホスティング or `dist/` をそのまま配布可能）。
  - 開発時のみ `vite dev` を使用。本番はビルド済み静的アセット。
- **言語/型**: TypeScript。オートマトンや字母テーブルの型安全性が高まり、リファクタや単体テストがやりやすい。
- **i18n**: [react-i18next](https://react.i18next.com/) + i18next（npm 依存）。翻訳リソースは `src/locales/ja.json` / `src/locales/ko.json`。
- **状態管理**: React の `useState`/`useReducer` + Context で十分。グローバル状態が増えたら Zustand を検討（v1 では React 標準で開始）。
- **スタイリング**: Tailwind CSS（Vite プラグイン経由）。ユーティリティクラスで構築し、テーマ（配色・キーサイズ等）は Tailwind の `theme` 設定 / CSS 変数で集中管理。レスポンシブはブレークポイントユーティリティで対応。`src/index.css` に Tailwind ディレクティブと最小限のグローバル定義を置く。
- **テスト**: Vitest（ロジック単体）。`automaton` の合成テストを中心に。
- **品質**: ESLint + Prettier、`tsc --noEmit` 型チェック。

> ハングル入力オートマトン・字母テーブル・統計計算などのコアロジックは **React 非依存の純粋な TS モジュール**として `src/core/` に置き、UI（React）から呼び出す。これによりテスト容易性とフレームワーク非依存性を確保する。

## 核心: ハングル入力オートマトン（`src/core/hangul/`）

韓国語入力は「物理キー → 字母 → 音節ブロック合成」の 3 段。OS の IME を使わず TS で再現する。
React 非依存の純粋関数群として実装し、`useHangulInput` フックから駆動する。

### 1. 物理キー → 字母 マッピング（2벌식, `layout.ts`）
QWERTY 物理キー位置に対応する字母（`KeyboardEvent.code` ベースで配列依存を回避）:

```
子音:  q ㅂ  w ㅈ  e ㄷ  r ㄱ  t ㅅ   a ㅁ  s ㄴ  d ㅇ  f ㄹ  g ㅎ
       z ㅋ  x ㅌ  c ㅊ  v ㅍ
母音:  y ㅛ  u ㅕ  i ㅑ  o ㅐ  p ㅔ   h ㅗ  j ㅓ  k ㅏ  l ㅣ
       b ㅠ  n ㅜ  m ㅡ
Shift: Q ㅃ  W ㅉ  E ㄸ  R ㄲ  T ㅆ   O ㅒ  P ㅖ
```

### 2. Unicode 音節合成（`jamoTables.ts`）
現代ハングル音節ブロックは `0xAC00 + (초성 × 588) + (중성 × 28) + 종성` で算出。
- 초성 19 個 / 중성 21 個 / 종성 28 個（index 0 = 받침なし）
- 結合母音（복합모음）: ㅗ+ㅏ=ㅘ, ㅗ+ㅐ=ㅙ, ㅗ+ㅣ=ㅚ, ㅜ+ㅓ=ㅝ, ㅜ+ㅔ=ㅞ, ㅜ+ㅣ=ㅟ, ㅡ+ㅣ=ㅢ
- 겹받침（複合終声）: ㄳ ㄵ ㄶ ㄺ ㄻ ㄼ ㄽ ㄾ ㄿ ㅀ ㅄ
- 분해表（겹받침・복합모음 → 構成字母）も持ち、終声の繰り上がり時に使う

### 3. 合成オートマトン（`automaton.ts`）★
韓国語 IME と同じ挙動を状態機械で再現する。状態 = 構築中の音節の `{초성, 중성, 종성}`。

入力字母が来たときの主な遷移:
- 母音が来た → 直前に確定しかけた **終声を「次の音節の初声」へ繰り上げ**（例: 「간」+ㅏ → 「가나」）。겹받침なら片方だけ繰り上げ。
- 子音が来た → 終声として付与可能なら付与。すでに終声があれば겹받침を試み、不可なら新しい音節を開始。
- 母音 + 母音 → 복합모음に結合可能なら結合、不可なら新音節。
- Backspace → 最後に追加した字母単位で 1 つ分解（IME 的な部分削除）。

出力は「確定文字列 + 合成中の 1 音節」。お題比較はこの合成結果に対して行う。

> 注: お題が既知文字列なので完全な IME 互換は必須ではないが、**実際の韓国語入力体験に近づけるため繰り上がり・結合は実装する**。jamoTables と分解表があれば自前実装は十分現実的。`useHangulInput` フックがこのオートマトンの状態を保持し、React 側へ「確定文字列＋合成中音節」を渡す。

## ゲームロジック

### モード（`core/game/modes.ts`, `data/`）
1. **音節モード**: 1 音節ずつ出題。初学者向け。お題データ = 頻出音節リスト。
2. **単語モード**: 基本語彙を出題。各語に「日本語の意味 / カナ読み」を併記（学習補助、i18n 連動）。
3. **文章モード**: 短い例文を出題。スペース・句読点も入力対象。

各モードは「出題リストからランダム or 順番に取り出す」「制限時間 or 問題数」で構成。

### 判定（`core/game/engine.ts`, `useGameEngine`）
- お題文字列とオートマトンの合成結果を文字単位で比較。
- お題表示は **確定済み(正)= 緑 / 合成中 = 下線 / 未入力 = 灰 / 誤入力 = 赤** で色分け（`Prompt.tsx`）。
- 1 問完了 → 次の問題へ。誤りは即時フィードバック（誤キーを赤フラッシュ）。

### 統計（`core/game/stats.ts`）
- **타수 (CPM, 打/分)**: 韓国語の標準指標。押下字母キー数 / 経過分。
- **WPM**: 参考値として（확정음절数 or 5 文字=1 語換算）。
- **正確度 (%)**: 正字母 / 総打鍵。
- 計測開始は最初の打鍵から。リアルタイム更新。

### スコア / タイマー（`core/game/score.ts`）
- 残り時間表示（モード設定の制限時間 or カウントアップ）。
- スコア = 正解音節 × 係数 + 速度ボーナス、連続正解で **コンボ倍率**。
- 終了時に結果サマリ（타수 / 正確度 / スコア）を表示。

## UI / 画面構成

1. **スタート画面**: モード選択（音節/単語/文章）、言語切替（日/한）、開始ボタン。
2. **プレイ画面**:
   - 上部: お題表示（色分け）＋ 統計バー（타수・正確度・残り時間・スコア・コンボ）
   - 下部: **仮想キーボード**（`Keyboard.tsx`）。各キーに字母ラベル、**次に押すキーをハイライト**、押下時にアニメーション。Shift で双子音/ㅒㅖ表示に切替。
3. **結果画面**: サマリ、再挑戦 / モード選択へ戻る。

### i18n（`src/i18n.ts`, `src/locales/`）
- react-i18next で `ja` / `ko` を切替。UI 文言（メニュー・統計ラベル・結果）を翻訳。
- 単語モードの「意味」は ja/ko で出し分け（韓国語 UI では韓国語の説明 or 非表示）。
- 言語選択は state に保持（v1 では保存しない＝リロードで既定言語に戻る）。

## アクセシビリティ / 対応環境
- 物理キーは `KeyboardEvent.code` で判定し、ユーザーの OS キーボード配列に依存しない。
- IME が ON だと `keydown` が横取りされるため、**入力欄では IME OFF 前提**（フォーカス時に注意表示、`inputmode`/`lang` 属性で配慮）。
- PC（物理キーボード）が主対象。モバイルは v1 では best-effort。
- 主要ブラウザ（Chrome/Firefox/Safari/Edge 最新）対応。

## 検証方法
- `automaton.test.ts`（Vitest）: `ㅎㅏㄴㄱㅡㄹ` → `한글`、繰り上がり `간`+`ㅏ`→`가나`、겹받침 `ㄺ`、복합모음 `ㅚ`、Backspace 分解などの代表ケース。
- `tsc --noEmit` + ESLint で型・静的チェック。
- `vite dev` で各モードを実際に打鍵し、色分け・統計・キーボードハイライトを目視確認。
- `vite build` → `vite preview`（or `dist/` を任意の静的サーバ/ローカルで開く）で、**ビルド済み静的ファイルのみで動作する**ことを確認。
